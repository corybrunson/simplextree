Simplextree

2019-05-08

simplextree is an R package aimed at simplifying computation for general simplicial complexes. This package facilitates this aim by providing an R-bindings to a Simplex Tree data structure, implemented using modern C++11 and exported as a Rcpp module.

The Simplex Tree data structure was originally introduced in the following paper:

Boissonnat, Jean-Daniel, and Clément Maria. “The simplex tree: An efficient data structure for general simplicial complexes.” Algorithmica 70.3 (2014): 406-427.

A Simplex Tree is an ordered, trie-like structure. Here’s a picture of a simplicial complex (left) and its corresponding Simplex Tree (right):

Installation

The current development version can be installed with the devtools package:

require("devtools")
devtools::install_gitub("peekxc/simplextree")

A stable CRAN release is planned for the future.

Quickstart

Create a simplex tree by calling simplex_tree()

library("simplextree")
st <- simplex_tree()

Specify simplices as vectors.

## Inserts the 2-simplex { 1, 2, 3 }. Also inserts its faces.
st$insert(c(1, 2, 3))
print(st)
## Simplex Tree with (3, 3, 1) (0, 1, 2)-simplices

Specify collections of simplices as lists of vectors

## Inserts two 0-simplices, { 4 } and { 5 }, and one 1-simplex, { 5, 6 }
st$insert(list(4, 5, c(5, 6)))
print(st)
## Simplex Tree with (6, 4, 1) (0, 1, 2)-simplices

Finding and removing simplices are handled identically.

st$find(c(2, 3)) ## TRUE
## [1] TRUE
st$remove(list(4, 5))
print(st)
## Simplex Tree with (4, 3, 1) (0, 1, 2)-simplices

A visual summary of the complex is available via the print_tree method

print(st) # also see $dimension and $n_simplices
## Simplex Tree with (4, 3, 1) (0, 1, 2)-simplices
st$print_tree()
## 1 (h = 2): .( 2 3 )..( 3 )
## 2 (h = 1): .( 3 )
## 3 (h = 0): 
## 6 (h = 0):

Querying the trie is handled largely through traversals. A traversal on a simplicial complex \(K\) is an ordered path through a subset \(S \subseteq K\). Traversals are specified with a simplex (optional), a unary function \(f\) (required), and a traversal type string (required). If no simplex is specified, the traversal begins at the empty_face.

For example, a depth-first traversal beginning at the empty face is achieved as follows: 

st$traverse(print, "dfs") # equivalent to st$traverse(empty_face, print, "dfs")
## numeric(0)
## [1] 1
## [1] 1 2
## [1] 1 2 3
## [1] 1 3
## [1] 2
## [1] 2 3
## [1] 3
## [1] 6

Or to start the traversal at a simplex:

## Prints the cofaces of { 2 }
st$traverse(2L, print, "cofaces")
## [1] 1 2
## [1] 1 2 3
## [1] 2
## [1] 2 3

Some traversals require additional parameters. These must be passed via a named list, e.g.

st$traverse(empty_face, print, "maximal-skeleton", list(k = 1L))
## [1] 1 2
## [1] 1 3
## [1] 2 3

The results can also be collected in a list, similar to lapply

## Get the simplices in the link of the 1-simplex { 2, 3 }
st$ltraverse(3, identity, "link")
## [[1]]
## [1] 1
## 
## [[2]]
## [1] 1 2
## 
## [[3]]
## [1] 2

Traversals can also be nested

## Prints the cofaces of the 0-simplices
st$traverse(empty_face, function(simplex){ 
  cofaces <- st$ltraverse(simplex, identity, "cofaces")
  cat(sprintf("Cofaces of %s: %s\n", simplex, paste0(cofaces, collapse = ", ")))
}, "maximal-skeleton", list(k = 0))
## Cofaces of 1: 1, c(1, 2), c(1, 2, 3), c(1, 3)
## Cofaces of 2: c(1, 2), c(1, 2, 3), 2, c(2, 3)
## Cofaces of 3: c(1, 2, 3), c(2, 3), c(1, 3), 3
## Cofaces of 6: 6

A variety of topological operations, such as elementary collapses and edge contractions, are supported.

## Collapse the free pair ({ 3 }, { 6 }) -> { 6 }
st$collapse(3, 6, 6)
## [1] TRUE
st$print_tree()
## 1 (h = 2): .( 2 6 )..( 6 )
## 2 (h = 1): .( 6 )
## 6 (h = 0):

Additionally, simple serialization and persistent storage interfaces are available as well:

st1 <- simplex_tree()
st2 <- simplex_tree()

## Inserts 11 simplices
st1$insert(list(1, 2, 3, c(1, 2), c(2, 3), c(1, 3), c(1, 2, 3), 4, 5, c(4, 5), 6))

## serialize exports the maximal faces needed to restore the complex 
st_serialized <- st1$serialize()
writeLines(paste0(st_serialized))
## c(1, 2, 3)
## c(4, 5)
## 6
## deserializing inserts the faces
st2$deserialize(st_serialized)
all.equal(st1, st2)
## [1] TRUE
